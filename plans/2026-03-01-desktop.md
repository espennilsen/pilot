# Desktop — Virtual Display for Agent

> Feature branch: `feature/docker-sandbox`
> Status: Implemented

---

## Overview

The "Desktop" feature provides project-scoped Docker containers with a virtual display (Xvfb + noVNC) that the agent can control programmatically via mouse, keyboard, screenshot, clipboard, and browser tools. Each desktop is tied to an active project and persists across sessions. Desktop tools are disabled by default to avoid wasting tokens.

**Key differentiator:** The agent can see and interact with a graphical desktop — useful for browser testing, GUI automation, visual verification, and any task that needs a screen.

## Settings

| Setting | Location | Default | Description |
|---|---|---|---|
| `desktopEnabled` | App settings | `false` | Show the Desktop tab in context panel |
| `desktopToolsEnabled` | Per-project `.pilot/settings.json` | inherits global | Override: enable/disable desktop tools for this project |

Toggling either setting hot-updates live agent sessions (tools are added/removed without starting a new chat).

## Custom Dockerfiles

Users can extend the base image per-project:

```
<project>/.pilot/desktop.Dockerfile
```

```dockerfile
FROM pilot-desktop:latest
RUN apt-get update && apt-get install -y python3
RUN pip install playwright && playwright install --with-deps
```

The project image is built automatically on desktop start, cached by a stable hash of the project path, and only rebuilt when the Dockerfile is newer than the cached image.

## Cross-Platform Docker Detection

`resolveDockerOptions()` probes for Docker in priority order:

1. `DOCKER_HOST` env var (unix://, npipe://, tcp://)
2. Windows: named pipe `//./pipe/docker_engine`
3. macOS/Linux: probe known socket paths:
   - `~/.docker/run/docker.sock` (Docker Desktop)
   - `/var/run/docker.sock` (Linux standard)
   - `~/.colima/default/docker.sock` (Colima)
   - `~/.rd/docker.sock` (Rancher Desktop)

Availability check returns context-specific messages ("Docker Desktop is installed but not running" vs "Docker is not installed") with a Retry button.
